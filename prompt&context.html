一、核心底层逻辑：AI效果的本质公式

总览

理解AI效果的本质，需要从两个维度切入：效果公式揭示AI输出的决定因素，表达三要素揭示人机沟通的信息传递机制。两者共同构成Prompt工程的底层理论基础。

1. 效果公式

AI效果 = LLM × (Task + Prompt)
  • LLM：大语言模型的基础能力（由参数量、训练数据、架构决定，是上限）
  • Task：明确的任务目标（用户的真实需求）
  • Prompt：将任务转化为模型可理解的指令，是释放模型能力的关键

LLM — 能力上限

由模型的参数量、训练数据规模与质量、网络架构决定。它是AI效果的「天花板」，决定了模型能理解多复杂的指令、能生成多高质量的内容。

Task + Prompt — 能力释放

即使拥有强大的LLM，如果Task模糊或Prompt低效，模型能力也无法充分释放。Prompt是连接用户意图与模型能力的「钥匙」。

核心结论：Prompt 是连接用户意图与模型能力的桥梁，Context 是 Prompt 的核心输入载体

2. 人机沟通的本质：表达三要素

表达 = 本意 + 文意 + 解意
  • 本意：用户的真实需求（由经验、词汇、知识构成）
  • 文意：Prompt 本身（编码过程，将本意转化为模型可理解的文本）
  • 解意：模型对 Prompt 的理解与执行（解码过程，最终输出结果）

编码过程：本意 → 文意

用户将脑海中的真实需求（本意）转化为文字描述（文意/Prompt）。这一步的质量取决于用户的表达能力、对任务的理解深度以及对模型工作机制的认知。

解码过程：文意 → 解意

模型读取Prompt（文意）后进行理解与执行，产生输出（解意）。模型的理解可能存在偏差，因此需要Prompt尽量精准、无歧义。

核心要求：Prompt 必须精准、无歧义，最大化「本意」与「解意」的一致性
实践启示：当AI输出不符合预期时，首先检查「本意→文意」的编码是否有信息丢失，其次检查「文意→解意」的解码是否存在歧义，而非简单归因于模型能力不足。

二、Prompt 工程核心原则与设计规范

总览

Prompt工程包含四大核心模块：基础表达原则（直接提问、正向描述）、结构化设计（标准模板与Markdown规范）、思维链引导（分步推理）、少样本提示（ICL上下文学习）。从原则到技术，层层递进。

1. 基础表达原则

（1）直接提问，删除冗余礼貌用语

核心逻辑：礼貌用语（请您、麻烦、辛苦、谢谢）会稀释核心指令，降低模型对任务的聚焦度
错误（冗余礼貌版） 正确（直接指令版）
❌请问你能不能告诉我2024年诺贝尔物理学奖的得主是谁呀？ ✅2024 年诺贝尔物理学奖得主
❌麻烦你帮我把这段文字翻译成英文可以吗？内容是:[文本] ✅翻译这段文字为英文:[文本]

（2）正向提问，避免负面词语

核心逻辑：大模型对「否定词」的理解存在偏差，负面指令会干扰模型对目标的聚焦
错误示例：别太长、不要不像人一样、不要太专业 ❌
正确示例：写一段护手霜的种草文案，要口语化，像和闺蜜聊天，控制在150字以内 ✅
核心要求：用正向描述明确目标，而非用否定词限制行为

2. 结构化 Prompt 设计

（1）标准通用模板（提示词库通用结构）

# 角色: 角色名称 角色概述和主要职责的一句话描述 ## 目标: 角色的工作目标（建议聚焦1-2个，可分点列出） ## 技能: 1. 为了实现目标，角色需要具备的技能1 2. 为了实现目标，角色需要具备的技能2 3. ... ## 工作流: 1. 描述角色工作流程的第一步 2. 描述角色工作流程的第二步 3. ...

适配场景

通用任务、角色扮演、知识库问答、插件调用等全场景

核心优势

层级清晰、无歧义，让模型精准理解角色、任务、输出要求

（2）Markdown 语法规范

核心价值：用层级结构（#/##）、列表、粗体等，强化指令的结构化，提升模型理解效率
# 角色:你是一位资深律师 ## 任务:分析以下合同条款的法律风险 1. 条款一:… 2. 条款二:… ## 输出要求: - 分点列出风险 - 重要风险用**粗体**标注 - 提供规避建议

3. 思维链引导：分步思考 Prompt

核心逻辑：引导模型按步骤推理，避免跳跃式输出，提升结果的逻辑性与准确性

示例：电商AI客服Prompt设计

任务: 设计电商AI客服的基础Prompt模板 请按以下步骤思考并输出: 1. 先梳理电商AI客服的3个核心用户场景(比如咨询物流、售后、商品参数) 2. 针对每个场景，梳理用户的核心需求(比如物流场景的需求是「快速知道物流状态」) 3. 基于场景和需求，设计对应的Prompt模板，每个模板不超过50字
适用场景：复杂任务、推理类任务、多步骤任务
实践技巧：在Prompt中明确写出「请按以下步骤思考并输出」，并编号列出每一步，模型会严格按步骤输出，减少遗漏和跳跃。

4. 少样本提示（ICL 上下文学习）

核心技术：ICL（In-Context Learning，上下文学习），大模型通过少量示例学习任务格式，无需微调
提示类型 示例数量 适用场景
Zero-Shot Prompting（零样本） 0个 简单、通用任务，模型已具备基础能力
One-Shot Prompting（单样本） 1个 格式要求明确，仅需1个示例即可对齐
Few-Shot Prompting（少样本） 2-5个 复杂格式、垂直领域任务，提升输出一致性
与涌现能力关联：少样本学习是大模型的核心涌现能力，参数量≥10B时性能大幅提升

Zero-Shot 适用情况

当任务足够通用（如翻译、摘要），模型预训练已覆盖该能力时，无需示例即可完成。优点是Prompt简洁，缺点是对格式控制较弱。

Few-Shot 适用情况

当任务格式复杂或属于垂直领域时，2-5个示例可显著提升输出的一致性和准确性。示例越多，格式对齐越精确，但Prompt也越长。

三、Context（上下文）核心体系

总览

Context是Prompt的核心输入载体，涵盖定义与作用、大模型流程中的演进（3.0→4.0）、窗口优化与增强技术（Query改写、多路检索）。理解Context体系，是掌握RAG和长上下文应用的关键。

1. Context 定义与核心作用

定义：模型生成回答时可参考的全部信息，包括历史对话、外部知识库、任务背景、用户画像等，是Prompt的核心输入

消解指代

补充多轮对话上下文，让模型理解「它」「这个」等指代内容。例如用户说「把它改一下」，Context中的历史对话可明确「它」指代什么。

知识增强

通过RAG注入外部知识库、业务文档，弥补模型预训练知识的时效性/垂直性不足。例如注入最新财报数据，让模型基于事实回答。

行为约束

通过历史对话保持角色一致性、输出风格一致性。例如前文设定了「专业律师」角色，Context确保后续回答保持专业口吻。

任务补全

提供任务背景、数据、规则，让模型精准理解任务边界。例如提供产品参数表，让模型基于真实数据生成推荐文案。

2. 大模型流程中的 Context 演进

（1）LLM 3.0 流程（无RAG）

Context Window → Query → System Prompt → ICL → Chat History → LLM → Summary → CoT草稿 → Response
Context 仅包含：上下文窗口、系统提示、少样本示例、聊天历史

（2）LLM 4.0 流程（RAG增强）

Context Window → Query → System Prompt → ICL → Chat History → Summary → CoT草稿 → Chunks → LLM → Response
核心升级：新增 Chunks（检索到的上下文片段），通过RAG将外部知识注入Context，实现「长期记忆」与「知识外挂」

3.0 的局限

Context仅来源于对话历史和系统提示，无法获取模型预训练数据之外的知识，对时效性信息和垂直领域知识支持不足。

4.0 的突破

通过RAG检索外部知识库，将相关Chunks注入Context，使模型能够基于最新、最专业的知识生成回答，突破了预训练数据的时效性限制。

3. Context 窗口与优化

Context Window（上下文窗口）：模型单次可处理的最大Token数量，是Context承载能力的上限
  • 核心影响：决定Prompt可注入的历史对话、外部知识、任务指令的总量，是长文档、多轮对话的核心瓶颈
  • 优化方向：上下文压缩、RAG检索、分层分块、Query改写，最大化有效Context占比

上下文压缩

对历史对话进行摘要压缩，保留关键信息，减少Token占用。例如将10轮对话压缩为1段摘要。

RAG检索

不将全部文档注入Context，而是按Query相关性检索最相关的片段，精准注入所需知识。

分层分块

将长文档按语义段落分块，按需加载相关块，避免一次性注入过多无关内容。

Query改写

优化查询语句，提升检索精准度，确保召回的Context与任务高度相关。

4. Context 增强技术

（1）Query 改写

目标

结合对话历史，将模糊查询转为语义完整的查询，消解歧义、补全信息

策略

事实补全、意图拆解、领域标签强化、上下文拼接

作用：提升Context的相关性与有效性，减少无效信息

（2）多路查询与检索

流程：多路召回相关Context → 结果合并 → 去重（以文档唯一ID为键）
  • 核心指标：召回率≥90%、准确率≥85%
  • 作用：为Prompt提供高质量、高相关的Context输入
多路召回的意义：单一检索方式可能遗漏相关内容，通过多种检索策略（关键词检索、语义检索、混合检索）并行召回，再合并去重，可最大化召回覆盖率，确保Context的完整性。

四象限模型总览

基于「用户已知/未知」×「AI已知/未知」，划分四类Prompt设计场景

📘 学习（左上）

场景定义：我不知道，AI知道

核心问题：如何提问？

Prompt 设计方向：开放式提问、引导式提问、少样本提示，挖掘AI知识

📗 助理（右上）

场景定义：我知道，AI知道

核心问题：如何下发指令？

Prompt 设计方向：结构化指令、正向描述、思维链引导，精准控制输出

📙 共创（左下）

场景定义：我不知道，AI不知道

核心问题：如何协作？

Prompt 设计方向：探索式Prompt、分步推理、多轮迭代，共同解决未知问题

📕 投喂（右下）

场景定义：我知道，AI不知道

核心问题：如何引导？

Prompt 设计方向：注入Context（知识库、规则、数据）、RAG增强、示例教学，让AI学习新知识

学习象限：我不知道，AI知道

核心问题：如何提问？

场景特征

用户对某个领域不了解，但AI在预训练中已掌握相关知识。典型场景如：学习新概念、了解行业知识、获取事实信息。

Prompt 设计方向

  • 开放式提问：「请介绍量子计算的基本原理和应用场景」
  • 引导式提问：「请从定义、核心技术和实际应用三个维度解释区块链」
  • 少样本提示：给出期望的回答格式示例，让AI按格式输出
关键技巧：用结构化的问题框架引导AI输出，避免过于宽泛的提问导致回答缺乏深度。例如「请从X、Y、Z三个角度分析」比「请介绍一下X」更有效。

助理象限：我知道，AI知道

核心问题：如何下发指令？

场景特征

用户明确知道自己的需求和期望输出，AI也具备完成该任务的能力。典型场景如：文案撰写、代码生成、数据整理、翻译润色。

Prompt 设计方向

  • 结构化指令：用标准模板（角色-目标-技能-工作流）精确描述任务
  • 正向描述：明确告诉AI「要做什么」，而非「不要做什么」
  • 思维链引导：对复杂任务添加分步推理要求
关键技巧：这是最高效的象限，Prompt的核心是「精准控制」。通过结构化模板+输出约束，确保AI的输出与预期高度一致。

共创象限：我不知道，AI不知道

核心问题：如何协作？

场景特征

用户和AI都缺乏完整答案，需要通过多轮交互共同探索。典型场景如：创新方案设计、复杂问题拆解、跨领域融合。

Prompt 设计方向

  • 探索式Prompt：「让我们头脑风暴，探讨如何将AI应用于教育领域」
  • 分步推理：将大问题拆解为小问题，逐步推进
  • 多轮迭代：基于AI的初步输出，追问、修正、深化，逐步逼近最优解
关键技巧：共创象限不追求一次性完美输出，而是通过「提问→反馈→追问→深化」的迭代循环，逐步构建解决方案。每一轮对话都是对Context的丰富。

投喂象限：我知道，AI不知道

核心问题：如何引导？

场景特征

用户掌握AI未知的专业知识、最新数据或特定规则，需要将这些信息注入AI。典型场景如：企业内部知识问答、基于最新数据的分析、特定规则下的任务执行。

Prompt 设计方向

  • 注入Context：将知识库、规则、数据直接写入Prompt或通过RAG检索注入
  • RAG增强：构建外部知识库，按需检索相关片段注入Context
  • 示例教学：通过Few-Shot示例，让AI学习特定格式和规则
关键技巧：投喂象限的核心是「Context质量」。注入的信息越精准、越结构化，AI的输出越可靠。RAG技术是此象限的核心工具。

五、辅助理论：乔哈里视窗与沟通优化

总览

乔哈里视窗将人际沟通信息分为四个象限：公开象限（双方都知道）、隐私象限（用户知道AI不知道）、盲点象限（AI知道用户不知道）、潜能象限（双方都不知道）。该模型可直接映射到Prompt优化中。

乔哈里视窗模型总览

核心逻辑：人际沟通的信息分为四个象限，可用于优化Prompt的信息传递效率
象限 定义 优化方向
公开象限
自己知道+他人知道		 双方都明确的信息 直接指令，减少冗余
隐私象限
自己知道+他人不知道		 用户明确但AI未知的信息 完整注入Context，补充背景知识

公开象限：自己知道 + 他人知道

定义：双方都明确的信息 — 用户知道自己要什么，AI也理解任务目标

在Prompt中的表现

当用户需求清晰、任务目标明确，且AI具备相关能力时，沟通处于公开象限。这是最高效的沟通状态。

优化方向

  • 直接指令：无需过多解释背景，直接下达清晰指令
  • 减少冗余：删除礼貌用语和重复描述，聚焦核心任务
  • 结构化输出：明确输出格式，让结果可直接使用
示例：「将以下5条客户反馈按产品功能分类，输出为表格，列名：功能类别、反馈内容、情感倾向」— 需求清晰，AI完全理解，直接执行。

隐私象限：自己知道 + 他人不知道

定义：用户明确但AI未知的信息 — 用户脑海中的背景知识、业务规则、隐含假设

在Prompt中的表现

用户「以为AI知道」但实际AI并不了解的信息，是最常见的沟通失误来源。例如行业术语、公司内部流程、个人偏好等。

优化方向

  • 完整注入Context：将背景知识、业务规则显式写入Prompt
  • 补充背景知识：不要假设AI了解你的业务，提供必要的上下文
  • 定义术语：对行业专有名词给出明确定义
常见错误：「帮我写一份Q3复盘报告」— AI不知道你们的Q3目标、业务数据、团队情况，输出必然空洞。
正确做法：「帮我写一份Q3复盘报告。背景：Q3目标是DAU增长20%，实际增长15%。核心数据：[附数据]。关注点：用户留存率下降的原因分析。」

盲点象限：自己不知道 + 他人知道

定义：用户未意识到但AI可识别的信息 — AI能发现用户忽略的角度、风险或需求

在Prompt中的表现

用户提出需求时可能遗漏重要维度，而AI基于广博的知识可以补充。例如用户要求写营销方案但未考虑合规风险。

优化方向

  • 引导AI主动反馈：在Prompt中添加「请指出我可能遗漏的重要方面」
  • 挖掘潜在需求：让AI反问澄清问题，帮助用户发现盲点
  • 风险提示：要求AI在执行任务时标注潜在风险和注意事项
示例Prompt：「请帮我设计一个用户增长方案。在输出方案前，请先列出你可能需要向我确认的3-5个关键问题，以确保方案符合我的实际情况。」

潜能象限：自己不知道 + 他人不知道

定义：双方都未知的信息 — 需要通过协作探索才能发现的新知识、新方案

在Prompt中的表现

面对全新领域或创新性任务，用户和AI都没有现成答案。需要通过多轮对话、分步探索来逐步构建解决方案。

优化方向

  • 共创式Prompt：明确表达探索意图，邀请AI共同思考
  • 分步探索：将大问题拆解为可逐步验证的小问题
  • 迭代深化：基于每轮输出追问和修正，逐步逼近答案
示例Prompt：「我想探索AI在心理健康领域的创新应用。请先列出3个可能的方向，我们逐一讨论可行性和潜在挑战，最终形成一个可落地的方案框架。」

底层公式与沟通本质

AI效果 = LLM × (Task + Prompt)

LLM决定能力上限，Task明确需求方向，Prompt是释放能力的关键桥梁。三者缺一不可。

表达 = 本意 + 文意 + 解意

本意是用户真实需求，文意是Prompt编码，解意是模型解码输出。Prompt工程的目标是最大化本意与解意的一致性。

关键洞察：当输出不符合预期时，按「本意→文意→解意」链路排查，定位信息丢失或歧义发生的环节。

Prompt 原则与 ICL 技术

Prompt 四大原则

1. 直接提问去礼貌

删除「请您」「麻烦」等冗余用语，聚焦核心指令

2. 正向提问避否定

用「要做什么」替代「不要做什么」，明确目标

3. 结构化分层级

用Markdown/标准模板定义角色、目标、技能、工作流

4. 分步引导促推理

添加分步思考要求，提升结果逻辑性

提示类型 示例数量 适用场景
Zero-Shot 0个 简单通用任务
One-Shot 1个 格式对齐
Few-Shot 2-5个 复杂格式、垂直领域

步骤 1-2：明确本意 + 正向指令

步骤1：明确本意

目标：拆解用户真实需求（Task），避免模糊
  • 问自己：我真正想要什么？
  • 将模糊需求转化为具体、可执行的任务描述
  • 区分「想要什么」和「不想要什么」，聚焦前者
模糊本意：「帮我写个方案」— 什么方案？给谁看？什么目的？
明确本意：「帮我写一份面向C端用户的AI产品推广方案，目标是提升注册转化率，预算5万以内」

步骤2：正向指令

目标：用直接、无否定的描述明确目标
  • 删除所有礼貌用语（请您、麻烦、辛苦）
  • 将「不要X」改写为「要Y」（Y是X的正向替代）
  • 用祈使句直接下达指令

步骤 3-4：结构化输入 + 补充Context

步骤3：结构化输入

目标：用Markdown/标准模板定义角色、目标、技能、工作流
# 角色: [角色名称] [角色概述] ## 目标: [1-2个核心目标] ## 技能: 1. [技能1] 2. [技能2] ## 工作流: 1. [步骤1] 2. [步骤2] 3. [步骤3]

步骤4：补充Context

目标：注入历史对话、知识库、背景数据
  • 历史对话：补充多轮对话的上下文，消解指代
  • 知识库：通过RAG检索注入相关外部知识
  • 背景数据：提供任务所需的数据、规则、约束条件
关键原则：Context越精准、越结构化，AI输出越可靠。避免注入大量无关信息，聚焦与任务直接相关的内容。

步骤 5-7：引导推理 + 示例对齐 + 输出约束

步骤5：引导推理

目标：添加分步思考要求，提升结果逻辑性

在Prompt中添加「请按以下步骤思考并输出」，将复杂任务拆解为有序步骤，避免模型跳跃式输出。

步骤6：示例对齐

目标：根据任务复杂度添加0-5个示例（ICL）
任务复杂度 推荐示例数
简单通用（翻译、摘要） 0个（Zero-Shot）
格式要求明确 1个（One-Shot）
复杂格式/垂直领域 2-5个（Few-Shot）

步骤7：输出约束

目标：明确格式、字数、风格等要求
  • 格式：表格、列表、段落、JSON等
  • 字数：具体数字（如「控制在300字以内」）
  • 风格：口语化、专业、学术、幽默等
  • 语言：中文、英文、中英混合等
完整示例：「请分析以下3款竞品的功能差异。输出为对比表格，列名：功能点、产品A、产品B、产品C。重点差异用粗体标注。控制在500字以内，使用专业但易懂的语言。」
